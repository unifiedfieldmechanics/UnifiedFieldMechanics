## 3. Workflow for Future Publishes

To ensure this index remains consistent and accessible as new pages are created:
1. **Filename/URL Policy**: All new content files must be published using the standard `.md` extension to guarantee RAG parser compatibility.
2. **Directory Update**: Every new article must be appended as a new row to the table in Section 2 above, mapping its raw GitHub URL and live HTML URL.
3. **Internal Links**: The raw markdown feeder button at the top of the new article must point to its respective raw GitHub `.md` URL, alongside the absolute README link.
